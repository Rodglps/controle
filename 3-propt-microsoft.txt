Contexto do Projeto
Criar um sistema chamado Controle de Arquivos, responsável por:

Coletar arquivos EDI de adquirentes via SFTP
Consumir APIs externas para captura de dados
Registrar, mover e armazenar arquivos em

AWS S3, ou
SFTP interno


Identificar cliente e layout baseado em:

Nome do arquivo
Regras configuradas em tabelas
Leitura dos primeiros 7.000 bytes (quando necessário)


Registrar toda rastreabilidade do processamento

Arquitetura do Sistema
O processamento é dividido em dois pods principais:
1. Pod Orquestrador
Responsabilidades:

Ler configurações dos SFTP (servidores, paths, paths_in_out)
Conectar aos SFTPs e listar arquivos
Registrar arquivos no banco (file_origin)
Publicar mensagens no RabbitMQ
(podendo enviar 1 arquivo ou lote)

2. Pod Processador
Responsabilidades:

Consumir mensagens RabbitMQ
Baixar arquivo via streaming do SFTP
Identificar:

Cliente
Layout


Upload via streaming para:

S3 ou
SFTP destino


Persistir informações nas tabelas:

file_origin
file_origin_client
file_origin_processing


Atualizar status e rastreabilidade
Aplicar regras configuradas de identificação
Garantir paralelismo seguro
