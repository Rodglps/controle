Como um arquiteto de software senior, revisor tecnico,
especialista em java, preciso que me auxilie na validação de uma infraestrutura e implementação de codigo java, 
surgerindo melhores práticas, padrões de projeto, arquitetura de software, segurança,
performance, escalabilidade, manutenibilidade, testabilidade, documentação e outros 
aspectos relevantes para garantir a qualidade e eficiência do sistema. A especificaçoes são as seguintes:

   1 - O sistema vai ser mono repositorio, ou seja, todo o codigo fonte, testes, documentaçao e outros arquivos relacionados ao projeto v
   ao ficar em um unico repositorio.

   2 - Precisamos da implementação do arquivos yml e docker composes para subir a infraestrutura em kubernetes e docker localmente, incluindo os pods, serviços, deployments, volumes e outros recursos necessarios para o funcionamento do sistema.
   E tambem do codigo java, estrutura de fila RabbitMQ, um banco de dados Oracle, sftp e AWS S3 localmente para testes e desenvolvimento. Para os serviços como s3
   , banco de daods oracle, rabbitmq e sftp podem ser usados imagens docker oficiais ou de terceiros confiaveis, mas a configuração e orquestração deve ser feita por nos, e o codigo java deve ser implementado por nos seguindo as melhores práticas e padrões de projeto.
  
   3 - Sobre o sistema, ele se chama controle de arquivos, é responsavel por toda a entrada
  de dados para a conciliação de cartão, ele se conecta a servidores sftp para obter arquvios EDI 
  das adquirentes de mercados e tambem consome APIs para capitra das informaçoes de cartoes gerando arquivos
  para serem processados, ele faz isso e tem como responsabilidade consumir e movimentar esses arquivos para um
  destino, esse destino pode ser um Bucket no S3 ou um SFTP interno, durante esse process ele tem 
  de identificar o dono do arquivo pelo nome do arquivo e o layout do arquivo pelo nome do arquvio ou 7000 bytes 
  iniciais do arquivo, e manter toda a rasteabilidaded do processo e do arquivo

  4 - Arquitetura segue o seguinte definição, será baseada em kubernets com filas RebbitMQ, entao segregamos
  as responsabilidades em pods dentro do kuberntes, eles tem cnectividade com banco de dados para 
  gerar todas as informaçoes de processo e consumo de setup necessario e orquestação via RebbitMq 
  para manter controle e paralelimos dentro do possivel, dividimos os pods por responsabilidade: 
    O primeiro é o orquestrador, ele precisa se conectar no banco de dados pegar as configuraçoes de sftp e pastas dos sftp(server, sever_paths, sever_paths_in_out), 
    Ele ira se conectar nos sftps listar os arquivos nas pastas, registrar esses arquivos no banco de dados(file_origin) e gerar mensgaens 
    contendo um ou mais arquivos no topico rabbitMQ.
    
    O segundo pod é ira consumir a fila rabbityMQ e sera responsavel por se conectar ao sftp, fisicamente pegar o arquivo e aplicar as 
    regras e indentificação de cliente(customer_identification, customer_identification_rule) e layout(layout, layout_identification_rule), ele deve pegar o arqvuivo usando streaming para controle do uso dos recurosos, e fara 
    upload do arquvo coma mesma estrategia, seja em um AWS s3 ou SFTP, a identificaçao do cliente acontecera atraves do nome do arquivo, 
    teremos tabelas onde as regra de identificação seram cadastradas, a idientificação do layout do arquivo sera feita da mesma forma, 
    onde teremos regras de identificação cadastradas em tabelas, para ambos os casos todas as regras tem de dar true para identificar 
    o cliente e o layout, Ele deve atualizar a tabela de arquvios, gerar dados nas tabelas de cliente e arquivos e tambem gerar dados 
    das tabelas de cliente arquivos (file_origin) e processamento.
